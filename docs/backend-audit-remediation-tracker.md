## Purpose

This is the cross-session execution tracker for the backend audit backlog. It is not a roadmap. It is the ranked implementation queue, with status fields, dependencies, and completion criteria that can be updated incrementally by any agent or engineer.

## How To Use

- `Status`: `todo` | `in_progress` | `blocked` | `done`
- `Owner`: person or agent currently driving the item
- Update this file in every PR that materially changes one of these items
- Do not add infrastructure-migration duplicates to the main backlog; put them only in the Phase 7 section

## Phase 7 Collision Review

These audit concerns are intentionally **not tracked in the main backlog** because `docs/phase-7-infrastructure-migration-plan.md` already owns them:

- SQLite production scalability / lock-contention as the long-term persistence strategy
- Shared runtime state across processes
- Durable queued execution / worker topology
- Worker backlog monitoring tied to Celery rollout
- Stage-specific DB/queue rollback mechanics

Phase 7 explicitly says it will **not** change extraction contracts, traversal semantics, or user-owned controls. That means all extractor, schema, traversal, hardcoded-hack, simplification, and most observability issues from the audit remain valid and stay in this tracker.
